# 模块规范模板

> 所有 MODULE.md 必须遵循此格式

```markdown
# {模块名}

> 版本：x.x.x
> 状态：active / inactive

## 职责
{一句话说明这个模块做什么}

## 输入
- {文件路径} — {说明}
- {文件路径} — {说明}

## 输出
- {文件路径} — {说明}

## 执行规则
1. {步骤1}
2. {步骤2}

## 依赖
- {模块名} — {为什么依赖}
- {外部系统} — {说明}
```

---

## 模块依赖定义规则

**依赖写法：**
- `依赖: 无（基础模块）` — 没有上游模块
- `依赖: {模块名}` — 依赖某个模块的输出

**示例：**
```
## 依赖
- feedback-collector — 需要 feedback/*.jsonl
- memory-layer — 需要 themes/ 和 hot.md
- 无（基础模块）
```

**判断标准：**
- 如果模块A需要读取模块B的输出 → A 依赖 B
- 如果模块可以独立运行 → 无依赖
